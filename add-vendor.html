ADD A
Vendor

Adding support for a new vendor takes about 10 minutes. All you need to edit is one file — vendors.js.

The core rule: The extension UI, download logic, and settings are all in popup.js. It never needs to change when you add a vendor. All vendor-specific logic lives in vendors.js — just add a new object and write a scraper function.
1

Open vendors.js

In the extension folder, open vendors.js. You'll see the VENDORS object at the top, with one entry per supported site.

2

Research the vendor site in DevTools

Before writing the scraper, spend 5 minutes on the vendor site's product page with Chrome DevTools open. You need to find where images come from.

🔍

Elements panel

Inspect a product image. Check its src, data-src, and any data-* attributes. This tells you which attributes hold the real image URL.

🌐

Network tab → Img filter

Filter by "Img" type. Scroll the page to trigger lazy loads. You'll see the exact CDN domain and URL pattern all product images share.

⌨

Console test

Try document.querySelectorAll('img').length to count images, and document.querySelectorAll('img')[0].src to inspect the first one.

📄

View Source → Ctrl+F

Search for the CDN domain name you found in Network tab. This tells you if image URLs are embedded in the HTML (easy to regex) or loaded dynamically.

3

Copy the template block into vendors.js

The template is included at the bottom of the VENDORS object in vendors.js as a comment. Copy it, uncomment it, and fill in your vendor's details.

📄 vendors.js — new vendor template
"your-vendor-domain.com": {
  name: "Your Vendor Name",
  icon: "🛋", // any emoji
  color: "#f59e0b", // accent color in the popup UI
  scraper: function() {
    const seen = new Set();
    const results = [];

    function add(url) {
      if (!url || seen.has(url)) return;
      // skip icons/logos
      if (/icon|logo|placeholder/i.test(url)) return;
      // must be an image file
      if (!url.match(/\.(jpg|jpeg|png|webp)(\?.*)?$/i)) return;
      seen.add(url); results.push(url);
    }

    // ——— Write your detection logic here ———

    // Standard: all img tags
    document.querySelectorAll("img").forEach(img => add(img.src));

    // CDN pattern (replace with real domain):
    // const cdn = /https:\/\/cdn\.yourvendor\.com\/[^\s"']+\.(jpg|png)/gi;
    // (document.body.innerHTML.match(cdn)||[]).forEach(u => add(u));

    return results;
  }
},
4

Write the scraper function

The scraper runs inside the vendor's page (injected via chrome.scripting), so you have full access to document and window. It must return an array of absolute image URL strings.

Use the pattern library below to pick the right detection method for what you found in DevTools:

🧩 Common detection patterns
// 1. Standard img tags (most sites)
document.querySelectorAll("img").forEach(img => {
  add(img.src);
  add(img.dataset.src); // lazy-load
  add(img.dataset.lazySrc);
});

// 2. Zoom / gallery data attrs
document.querySelectorAll("[data-zoom-image],[data-large]")
  .forEach(el => add(el.getAttribute("data-zoom-image")));

// 3. CDN domain regex (fastest for most vendor portals)
const pattern = /https:\/\/cdn\.site\.com\/[^\s"'<>]+\.(jpg|png)/gi;
(document.body.innerHTML.match(pattern) || []).forEach(u => add(u));

// 4. CSS background-image on elements
document.querySelectorAll("[style*='background']").forEach(el => {
  const m = (el.getAttribute("style")||"")
    .match(/url\(['"]?([^'")\s]+)['"]?\)/i);
  if (m) add(m[1]);
});

// 5. Plain text URLs in a named section
const section = findByText("DOWNLOADS");
if (section) section.innerText.split("\n").forEach(line => {
  if (/^https?:\/\/.+\.(jpg|png)/i.test(line.trim())) add(line.trim());
});
Important: The scraper function is serialized as a string and injected into the page. This means it cannot reference variables or functions from outside itself — everything it needs must be defined inside the function body. The add() helper must be defined inside scraper().
5

Test in Chrome

Reload the extension (chrome://extensions → click the reload icon), navigate to the vendor's product page, and click Scan. Check the count — if it seems low, open DevTools console on that page and run parts of your scraper manually to debug.

⚠ If you get 0 results: The most common cause is that images are loaded dynamically by JavaScript after page load. Try scrolling the page first to trigger lazy loading, then scan again. If images only appear after user interaction (e.g. clicking a gallery), the scraper may need to wait — or you need to interact with the page first.
6

Reload the extension — done

No rebuild, no compile step, no changes to any other file. The extension reads vendors.js at runtime, so your new vendor appears in the popup and the settings vendor list automatically.

Quick reference

Detection Pattern Library

Match your DevTools finding to the right code pattern.

Pattern When to use Example
img[src] Images are standard <img> tags, not lazy-loaded Most basic product pages
img[data-src] Images use lazy loading — src is empty or a placeholder on load Sites using LazyLoad.js, Intersection Observer
data-zoom-image Product page has a zoom/lightbox gallery Magento, WooCommerce product pages
CDN regex Images all come from a single CDN domain you found in Network tab portal.vendor.com, cdn.vendor.com, images.vendor.com
background-image Product "images" are CSS backgrounds on div elements, not img tags Custom-built catalog pages, slide backgrounds
plain text URLs A "Downloads" or "Assets" section lists raw URLs as page text Wholesale portals, dealer sites
a[href] images Images are linked directly with anchor tags Simple catalog pages, press asset pages
