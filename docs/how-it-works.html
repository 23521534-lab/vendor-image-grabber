HOW IT
Works

From opening the extension to files landing in your Downloads folder — here's exactly what happens at each step.

Step-by-step flow
1

Step 01

Navigate to a vendor page

Go to any supported vendor's product catalog, category page, or individual product page in Chrome. The extension constantly monitors the active tab's URL and auto-detects when you're on a supported site.

Supported sites

🪞 Bassett Mirror bassettmirror.com
🏠 Worldwide Home worldwidehomefurnishingsinc.com
2

Step 02

Click the extension icon & hit Scan

The extension popup opens, showing the detected vendor. Click Scan for Images — this injects a scraper script directly into the live page using Chrome's chrome.scripting API.

4 detection methods run in parallel

🏷 img[src] + lazy attrs src, data-src, data-lazy-src, data-original
🔢 Data attributes data-zoom-image, data-large, data-full, data-img
🎨 CSS background-image Parses inline style attributes for url() values
🔗 CDN regex scan Scans raw HTML for image CDN URLs by pattern
3

Step 03

Review the thumbnail grid

All found images load as thumbnails in the extension popup. By default, all images are pre-selected. You can click individual thumbnails to toggle them, or use Select All to toggle everything. Use the format filter (All / JPG / PNG / WEBP) to narrow the selection.

4

Step 04

Download to your machine

Click Download Selected. The extension uses the native chrome.downloads API to save each file. Images are throttled with a 280ms delay between downloads to avoid browser queue limits and saved to a named subfolder:

Extension architecture

File Structure

The extension is 4 files. Vendor logic is fully separated from the UI engine.

Config

manifest.json

Declares the extension's permissions, entry points, and supported host patterns.

Vendors

vendors.js

The vendor registry. All scrapers and site configs live here. Add a new vendor here — nothing else needs to change.

UI Engine

popup.js + popup.html

Renders the extension popup, handles user interaction, triggers scraping, and manages downloads. Reads from vendors.js.
